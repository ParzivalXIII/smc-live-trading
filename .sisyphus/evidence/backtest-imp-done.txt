Two-Phase Replay Design
-----------------------

Phase 1 (Streaming — exact live path):
  for i in range(len(data)):
      result = engine.update(i, row)
      strategy_callback.update(i, row, result)
      record event if swing confirmed

Phase 2 (Batch — post-hoc analysis):
  bos_choch(data, swings)
  ob(data, swings)
  liquidity(data, swings)
  retracements(data, swings)
  → merge into 26-column per_candle_report


Event-Driven Core (V1/V2 Interface)
------------------------------------

for candle in candles:
    events = engine.update(candle)       # Phase 1
    strategy.update(candle, events)      # V2: trade sim plugs in here

Usage
-----

Python API:
  from backtest import BacktestConfig, BacktestHarness
  cfg = BacktestConfig(swing_length=5, confirmation_bars=2)
  harness = BacktestHarness(cfg)
  result = harness.run("data/EURUSD_15M.csv")
  result.metrics   → dict of 18 metrics
  result.report    → 26-column DataFrame
  result.events    → event log DataFrame

CLI:
  python backtest.py \
      --data tests/test_data/EURUSD/EURUSD_15M.csv \
      --output-dir ./results \
      --swing-length 5 \
      --confirmation-bars 2 \
      --close-break \
      --verbose
