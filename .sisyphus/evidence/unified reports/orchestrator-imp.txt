LiveSmcBuffer (live_smc_buffer.py)
------------------------------------

  Streaming SMC accumulator. Owns _SwingEngine + StructureEngine.
  Runs batch OB/liquidity/retracements ONLY on swing confirmation.

  Key design:
    - Every candle: _SwingEngine.update() → result dict
    - On swing confirmed: StructureEngine.update() + _recompute_downstream()
    - Every candle: StructureEngine.check_confirmations()
    - Report updated incrementally, trimmed to 200 rows

  26-column report schema:
    Timestamp, Open, High, Low, Close, Volume
    SwingHighLow, SwingLevel, SwingPivotIndex
    BOS, CHOCH, BOSLevel, BrokenIndex
    OB, OBTop, OBBottom, OBVolume, OBMitigatedIndex, OBPct
    Liquidity, LiqLevel, LiqEnd, LiqSwept
    RetraceDirection, CurrentRetracement%, DeepestRetracement%

  Column names match what SnapshotBuilder.build() expects.


Orchestrator (orchestrator.py)
-------------------------------

  State machine that owns the pipeline. Pure orchestration —
  no business logic, no scoring, no strategy.

  States:
    IDLE → LOAD → ANALYZE → DECIDE → JOURNAL → IDLE
                                            ↘
                                            ERROR (on exception)

  OrchestratorContext:
    symbol, timeframe, data_dir, db_path, mode ("live"/"replay")
    Runtime: ta_row, smc_report, snapshot, confluence, narrative,
             decision, entry (All None after reset())

  _TRANSITIONS matrix validates all state transitions.
  Invalid transitions raise RuntimeError.

  Pipeline steps:
    load()      — fetch TA data (or use pre-populated in replay mode),
                  update LiveSmcBuffer
    analyze()   — SnapshotBuilder.build() → ConfluenceScorer.score()
                  → MarketNarrativeBuilder.build()
    decide()    — DecisionEngine.decide()
    journal()   — Build JournalEntry (persisted by caller)

  step(): The one public method. Called by live or replay loop.
    On failure: sets state to ERROR, stores _last_error, re-raises.
    Caller must call reset() before retry.

    reset(): Clears all runtime context, sets state to IDLE.
             Works from ANY state (including ERROR).
             Bypasses _transition() intentionally.


Gap Solutions (Oracle-reviewed)
--------------------------------

  G1 — Replay mode:
    OrchestratorContext.mode: str = "live"
    load() returns early when mode == "replay"
    Caller pre-populates ta_row and smc_report

  G2 — Error recovery:
    _TRANSITIONS dict validates all 6 states
    _transition() raises RuntimeError on invalid
    reset() clears runtime context, works from any state
    step() has NO auto-recovery — caller owns retry policy

  G3 — Async bridge:
    sync_write_entry(writer, entry) wraps asyncio.run()
    Orchestrator stays pure synchronous
    Caller pattern: orch.step(); sync_write_entry(writer, entry)

Usage
------

  from orchestrator import LiveOrchestrator, OrchestratorContext, sync_write_entry
  from live_smc_buffer import LiveSmcBuffer
  from journal import JournalWriter

  # Live mode
  ctx = OrchestratorContext(symbol="BTCUSDT", timeframe="1d")
  buf = LiveSmcBuffer()
  orch = LiveOrchestrator(ctx, smc_buffer=buf)

  async def main():
      async with JournalWriter("journal.db") as writer:
          while True:
              try:
                  orch.step()
                  sync_write_entry(writer, orch.context.entry)
              except Exception:
                  if orch.state == OrchestrationState.ERROR:
                      orch.reset()
                  # Wait for next candle tick
