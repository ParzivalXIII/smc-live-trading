Architecture
-------------

Three new layers on top of existing MarketSnapshot + Confluence:

  MarketSnapshot
       ↓
  ConfluenceScorer    →  ConfluenceResult (bias, score, reasons)
       ↓
  MarketNarrativeBuilder  →  MarketNarrative (4 sections + conclusion)
       ↓
  DecisionEngine          →  Decision (action, confidence, invalidation, target)
       ↓
  (Human Decision Support — NOT trading)

  All layers share the same bias. Pipeline coherence verified end-to-end:
  SnapshotBuilder → ConfluenceScorer → MarketNarrativeBuilder → DecisionEngine

MarketNarrative (narrative.py)
-------------------------------

  @dataclass
  class NarrativeSection:
      title: str
      bullets: list[str]

  @dataclass
  class MarketNarrative:
      symbol: str
      timeframe: str
      bias: str
      score: int
      max_score: int
      sections: list[NarrativeSection]   # 4 sections: Trend, Momentum, Structure, Liquidity
      conclusion: str

  class MarketNarrativeBuilder:
      def build(snapshot, confluence) -> MarketNarrative

  Sections:
    Trend:      Price vs EMA21 (above/below + %), EMA21 slope direction
    Momentum:   MACD vs signal, RSI value + zone label (rsi_label()),
                MFI value + zone label (mfi_signal())
    Structure:  Last swing direction + level, BOS direction + status,
                CHOCH detection (or "No recent signals")
    Liquidity:  Nearest zones above/below with price levels, active OBs

  Conclusion templates:
    Bullish: "Bullish continuation favored while EMA21 remains intact.
              Liquidity target at {level}."
    Bearish: "Bearish momentum intact. Invalidation at {level}
              (nearest buy-side liquidity)."
    Neutral: "Mixed signals across timeframes. No clear directional edge.
              Stand aside."

  Fallback text when liquidity data is missing:
    "No clear target level identified." / "No clear invalidation level identified."


Decision Engine (decision_engine.py)
--------------------------------------

  @dataclass
  class Decision:
      bias: str
      confidence: float          # 0.0 to 1.0
      action: str                # "look_for_longs" / "avoid_shorts" / "stand_aside"
      invalidation: float | None # Liquidity-first, 3-level fallback
      target: float | None       # Liquidity-first
      breakout_pending: bool = False   # Modifier flag (not action)
      breakout_level: float | None = None

  Actions (from score):
    >= 7   → look_for_longs
    4 - 6  → stand_aside + breakout_pending=True
    0 - 3  → stand_aside
    < 0    → avoid_shorts

  Confidence: abs(score) / max_score

  Invalidation (liquidity-first):
    Bullish → nearest_liquidity_below → last_swing_low → EMA21 → None
    Bearish → nearest_liquidity_above → last_swing_high → EMA21 → None
    Neutral → None

  Target (liquidity-first):
    Bullish → nearest_liquidity_above
    Bearish → nearest_liquidity_below
    Neutral → None


Hierarchical MTF (confluence.py update)
----------------------------------------

  Widen ConfluenceResult.score from int to float.

  hierarchical composite_score():
    HTF (daily > h4 > h1) sets bias unconditionally.
    LTF adjusts score by ±30% (cannot flip bias).

    Agreement:   score += abs(ltf_score) * 0.3
    Disagreement: score -= abs(ltf_score) * 0.15   (asymmetric — see Deviation)

    Returns ConfluenceResult with HTF bias, float score.

  regime_alignment property:
    "aligned" — all TFs agree with HTF
    ["h4_conflict", "h1_conflict"] — list of conflicting TFs
    "neutral_if_no_htf" — no HTF available

  legacy_composite_score() preserved for backward compat.


KNOWN DEVIATION: Asymmetric LTF Disagreement Multiplier
---------------------------------------------------------

  The plan specifies ±30% for both agreement AND disagreement:

    Agreement:   score += abs(ltf_score) * 0.3
    Disagreement: score -= abs(ltf_score) * 0.3

  The implementation uses:

    Agreement:   score += abs(ltf_score) * 0.3    ✅ matches plan
    Disagreement: score -= abs(ltf_score) * 0.15  ⚠️ differs from plan

  Impact: Disagreement penalizes confidence half as much as agreement rewards it.
  This creates an asymmetric confidence adjustment where:

    - 2 agreeing LTFs + 0 disagreeing: full boost (+6.0 from two scores of 10)
    - 1 agreeing + 1 disagreeing: net +1.5 (3.0 - 1.5)
    - 0 agreeing + 2 disagreeing: net -3.0 (0.0 - 3.0)

  Whether intentional or accidental is unresolved. The implementation is
  self-consistent and all tests pass. The 0.15 multiplier produces softer
  disagreement penalties, which may be desirable (LTFs should refine, not
  overpower HTF). If full symmetry is desired, change line 312 in
  confluence.py from 0.15 to 0.3.

Usage
------

  from market_snapshot import MarketSnapshot, SnapshotBuilder
  from confluence import ConfluenceScorer, MarketContext
  from narrative import MarketNarrativeBuilder
  from decision_engine import DecisionEngine

  # 1. Build snapshot
  builder = SnapshotBuilder()
  snapshot = builder.build("BTC/USDT", "1d", ta_row, smc_report)

  # 2. Score it
  result = ConfluenceScorer().score(snapshot)

  # 3. Generate narrative
  narrative = MarketNarrativeBuilder().build(snapshot, result)
  for section in narrative.sections:
      print(f"\n{section.title}:")
      for bullet in section.bullets:
          print(f"  • {bullet}")
  print(f"\n{result.bias} Score: {result.score}/{result.max_score}")
  print(narrative.conclusion)

  # 4. Make decision
  engine = DecisionEngine()
  decision = engine.decide(snapshot, result)
  print(f"{decision.action} (confidence={decision.confidence:.2f})")
  print(f"  Invalidation: {decision.invalidation}")
  print(f"  Target: {decision.target}")
  print(f"  Breakout pending: {decision.breakout_pending}")
