Task 1: Single Orchestrator Instance Replay
---------------------------------------------

  Status: ✅ COMPLETE

  The replay mode was fixed to use a single orchestrator instance across the
  entire run instead of recreating ctx/orch per candle.

  The key fix: LiveOrchestrator.load() now updates the SMC buffer in BOTH
  live and replay modes. Previously, replay mode returned early before
  _smc_buffer.update(), causing every step to see identical buffer state.

  Correct replay pattern:
    ctx = OrchestratorContext(symbol, timeframe, mode="replay")
    buf = LiveSmcBuffer()
    orch = LiveOrchestrator(ctx, smc_buffer=buf)
    
    for i in range(len(data)):
        ctx.ta_row = ta_data.iloc[i]
        ctx.smc_report = smc_report
        orch.step()  # Buffer advances each call

  One orchestrator. One buffer. One continuous run.

Task 3: Live Loop Journal Writes
----------------------------------

  Status: ✅ COMPLETE

  Every completed step() produces a JournalEntry. The caller persists
  it to SQLite via sync_write_entry(). No gating — every cycle recorded.

  Pattern:
    async with JournalWriter("journal.db") as writer:
        while True:
            try:
                orch.step()
                sync_write_entry(writer, orch.context.entry)
            except Exception:
                if orch.state == OrchestrationState.ERROR:
                    orch.reset()

  sync_write_entry() wraps asyncio.run(writer.append() + writer.flush()).
  Orchestrator stays pure synchronous — no async leak into the pipeline.


Task 4: Thin Alert Layer on Top of Decision
---------------------------------------------

  Status: ✅ DESIGN COMPLETE (not yet implemented)

  Design spec for alerts.py:

    @dataclass
    class Alert:
        alert_type: str   # "bias_change", "confidence_change",
                          # "action_change", "breakout"
        severity: str     # "info", "warning", "critical"
        message: str
        previous: Any
        current: Any

    class AlertWatcher:
        def __init__(self):
            self._previous: Decision | None = None
        
        def check(self, decision: Decision) -> list[Alert]:
            alerts = []
            if self._previous:
                if decision.bias != self._previous.bias:
                    alerts.append(Alert("bias_change", "warning", ...))
                if decision.action != self._previous.action:
                    alerts.append(Alert("action_change", "info", ...))
                if (decision.breakout_pending and 
                    not self._previous.breakout_pending):
                    alerts.append(Alert("breakout", "info", ...))
            self._previous = decision
            return alerts

  Alerts fire on Decision field changes, not raw indicators.
  Decision already synthesizes all pipeline outputs (bias, confidence,
  action, invalidation, target, breakout_pending).

  Standalone module — orchestrator stays pure orchestration.
  Confidence thresholds (0.7 high, 0.3 low) configurable from start.

  Not yet implemented — ready for next sprint.
