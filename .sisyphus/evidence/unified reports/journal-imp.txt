Purpose
-------

The Journal is an append-only, queryable record of every decision cycle.
It answers: what the engine saw, what it scored, what it decided, and
what events supported it. No scoring, no strategy logic, no backtest
internals — pure storage.

JournalEntry (journal.py)

  @dataclass
  class JournalEntry:
      run_id: str                   # Compound key: {symbol}_{timeframe}_{ts}_{uuid}
      timestamp: pd.Timestamp
      symbol: str
      timeframe: str
      close: float
      direction_score: float
      bias: str
      confidence: float
      narrative_summary: str
      decision_action: str
      decision_invalidation: float | None
      decision_target: float | None
      breakout_pending: bool

JournalWriter (journal.py)
---------------------------

  class JournalWriter:
      async def __aenter__(self) -> JournalWriter    # Creates tables
      async def __aexit__(self, *args) -> None        # Flushes + closes
      async def append(self, entry: JournalEntry)     # Inserts run + events
      async def flush(self) -> None                   # Commits transaction
      async def query_runs(symbol, timeframe, limit)  # Returns list[dict]


SQLite Schema
--------------

  journal_runs:
    run_id              TEXT PRIMARY KEY     compound key
    timestamp           TEXT NOT NULL
    symbol              TEXT NOT NULL
    timeframe           TEXT NOT NULL
    close               REAL
    direction_score     REAL
    bias                TEXT
    confidence          REAL
    narrative_summary   TEXT
    decision_action     TEXT
    decision_invalidation REAL
    decision_target     REAL
    breakout_pending    INTEGER
    created_at          TEXT   DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
    
    INDEX idx_runs_symbol_timeframe ON (symbol, timeframe)
    INDEX idx_runs_timestamp ON (timestamp)

  journal_events:
    id                  INTEGER PRIMARY KEY AUTOINCREMENT
    run_id              TEXT NOT NULL  → FK → journal_runs(run_id)
    event_id            TEXT NOT NULL
    event_type          TEXT
    direction           INTEGER
    status              TEXT
    level               REAL
    event_timestamp     TEXT
    
    INDEX idx_events_run_id ON (run_id)
    INDEX idx_events_event_id ON (event_id)


Key Design Decisions
---------------------

  Storage:    SQLite via aiosqlite (async, non-blocking)
  Format:     Two tables (runs + events) with FK link
  run_id:     Compound key: {symbol}_{timeframe}_{YYYYMMDDTHHMMSS}_{8-char-uuid}
  NaN:        float('nan') → NULL in SQLite via _safe() helper
  Bool:       breakout_pending stored as INTEGER (0/1)
  Timestamps: UTC ISO 8601 format throughout
  Events:     Full StructureEvent objects embedded in JournalEntry,
              not just event_ids (cleaner append interface)


Momus Issues Addressed

  2. NaN close → _safe() converts to NULL

Usage
------

  from journal import JournalEntry, JournalWriter, make_run_id
  from smartmoneyconcepts.structures import StructureEvent
  import asyncio, pandas as pd

  async def log_decision(snapshot, confluence, narrative, decision, events):
      entry = JournalEntry(
          run_id=make_run_id(snapshot.symbol, snapshot.timeframe, snapshot.timestamp),
          timestamp=snapshot.timestamp,
          symbol=snapshot.symbol,
          timeframe=snapshot.timeframe,
          close=snapshot.close,
          direction_score=confluence.direction_score,
          bias=confluence.bias,
          confidence=confluence.confidence,
          narrative_summary=narrative.conclusion,
          decision_action=decision.action,
          decision_invalidation=decision.invalidation,
          decision_target=decision.target,
          breakout_pending=decision.breakout_pending,
          events=events,
      )
      async with JournalWriter("journal.db") as writer:
          await writer.append(entry)
          await writer.flush()

  # Query back
  async def get_recent(symbol="BTCUSDT"):
      async with JournalWriter("journal.db") as writer:
          return await writer.query_runs(symbol=symbol, limit=20)

  asyncio.run(log_decision(...))
