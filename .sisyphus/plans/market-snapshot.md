## Work Objectives

### Core Objective
Build a clean separation between "what the market IS" (MarketSnapshot) and "what the market MIGHT DO" (ConfluenceResult). The snapshot is pure state, the confluence is additive opinions with reasons.

### Concrete Deliverables
1. `market_snapshot.py` — `MarketSnapshot` dataclass (dataclass, slots=True) + `SnapshotBuilder.build()` method
2. `confluence.py` — `ConfluenceResult` dataclass + `ConfluenceScorer` class with additive scoring
3. `MarketContext` dataclass wrapping 3 timeframes + alignment detection
4. `tests/test_market_snapshot.py` — unit tests for all components + integration test with real data

### Production/Streaming Scope
SnapshotBuilder is designed for historical/batch analysis using the full 26-column per_candle_report from backtest.py. Integration with the streaming event pipeline (StructureEngine → smc_report construction) is deferred to a separate task. For live use, the TA data comes from `load_ta_latest()`, and the SMC report is generated by running the batch indicators on accumulated swing data.

  **Additive Scoring Table** (exact from spec):
  | Condition | Points | Max |
  |-----------|--------|-----|
  | `close > ema21` | +2 | — |
  | `ema21_slope > 0` | +1 | — |
  | `macd > macd_signal` | +1 | — |
  | `rsi14 > 55` | +1 | — |
  | `mfi14 > 50` | +1 | — |
  | `last_bos_direction == 1` | +3 | — |
  | `last_bos_direction == -1` | -3 | — |
  | `nearest_liquidity_above exists` | +1 | — |
  | `nearest_liquidity_below exists` | -1 | — |
  | **Total range** | — | **-4 to 10** |

  The minimum score (-4) occurs when all bullish conditions are false, BOS is bearish (-3), and bearish liquidity exists (-1). The maximum (10) occurs when all bullish conditions fire and no bearish conditions are active.

  **Score → Bias mapping**:
  - `< 0`: bearish
  - `0–3`: bearish
  - `4–6`: neutral
  - `7–10`: bullish

  **Reasons are MANDATORY**: Every point (positive or negative) must include a reason string describing WHY the point was awarded. Example reasons:
  - "close (50000.0) > ema21 (49000.0): +2"
  - "last_bos_direction == 1: +3 (bullish BOS detected)"
  - "nearest_liquidity_below exists at 48000: -1"

  The `score()` method must handle `None` values gracefully (e.g., if `last_bos_direction` is None, no points awarded for that condition).

  **Liquidity scoring note**: Both liquidity conditions (`nearest_liquidity_above` and `nearest_liquidity_below`) can fire simultaneously, netting to 0 (+1 from above, -1 from below). This is intentional — both conditions are independent factual observations. The scorer doesn't pick a direction; it reports what exists. The trader sees both reasons in the output.

- [ ] **T5. Create MarketContext with alignment detection**
  **What to do**: In `confluence.py` (or a new `market_context.py` — PREFER `confluence.py` to keep context together with scoring), add the following after ConfluenceScorer:

  ```python
  @dataclass
  class MarketContext:
      daily: MarketSnapshot | None = None
      h4: MarketSnapshot | None = None
      h1: MarketSnapshot | None = None

      def alignment(self) -> str:
          """Return 'bullish', 'bearish', or 'mixed' based on alignment of all non-None entries."""
          ...

      def composite_score(self) -> ConfluenceResult:
          """Score each snapshot individually and combine. Max score = 30."""
          ...
  ```

  **Alignment detection logic**:
  - Count non-None timeframes. If 0 → return "neutral"
  - If only 1 timeframe is non-None → return that timeframe's bias
  - If all non-None timeframes have the same bias → return that bias ("bullish" / "bearish")
  - If there's any disagreement among non-None timeframes → return "mixed"

  **Composite scoring**:
  - Score each snapshot with ConfluenceScorer
  - Sum scores. Normalize max_score to active timeframes: `active_timeframes = [tf for tf in [self.daily, self.h4, self.h1] if tf is not None]` and `max_score = len(active_timeframes) * 10`. If 0 active timeframes, max_score = 0.
  - Aggregate bias: same mapping as single (0-3 per-TF → bearish, etc.), BUT only if all align. If alignment is "mixed", keep individual scores but overall bias = "mixed".

  **Must NOT do**:
  - ❌ Do NOT store trading state or positions
  - ❌ Do NOT trigger external events or webhooks
  - ❌ Do NOT add persistence/database layer
